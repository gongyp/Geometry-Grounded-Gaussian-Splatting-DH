# 像素到高斯映射方法总结

本文档总结从 2D 图像像素反推对应 3D Gaussian 高斯点的各种方法，及其在增量更新场景下的适用性分析。

---

## 问题背景

在增量更新（Incremental Update）场景中，需要：
1. 对比渲染图像和真实图像，检测变化区域
2. 将 2D 变化掩码提升到 3D 空间
3. 识别哪些高斯对应变化区域，以便更新

核心问题：**如何建立 2D 像素与 3D 高斯之间的映射关系？**

---

## 方法总结

| 方案 | 名称 | 准确性 | 速度 | 实现难度 |
|------|------|--------|------|---------|
| 1 | 渲染器可见性 | 低 | 快 | 低 |
| 2 | 唯一ID编码 | 高 | 慢 | 中 |
| 3 | 解析贡献度计算 | 高 | 中 | 中 |
| 4 | 差分反向传播 | 中 | 中 | 中 |
| 5 | 尺度校准的深度估计 | 中 | 快 | 低 |
| 6 | 椭圆投影覆盖法 | 高 | 中 | 中 |

---

## 方案 1：渲染器可见性

### 原理
Gaussian 渲染器在光栅化时输出每个高斯的可见性信息 `radii > 0`，表示该高斯是否在相机视角下可见。

### 流程
```
渲染图像 → 获取 visibility_filter (radii > 0) → 得到可见高斯列表
```

### 优点
- 实现简单，直接利用渲染器输出
- 速度快

### 缺点
- **只能判断高斯是否可见，无法知道像素级对应关系**
- 变化区域可能只是某个可见高斯的一小部分

### 适用场景
仅需粗略确定哪些高斯需要更新时使用。

---

## 方案 2：唯一ID编码

### 原理
为每个高斯分配唯一的颜色标识，渲染后解码颜色获取高斯ID。

### 流程
1. 备份高斯原始颜色
2. 将每个高斯的颜色编码为唯一ID（如 RGB 256³ ≈ 16.7M 个）
3. 渲染图像
4. 解码图像中每个像素的颜色 → 得到贡献的高斯ID列表
5. 恢复原始颜色

### 优点
- **精确到像素级**，无歧义
- 数学上严格保证

### 缺点
- 需要多次渲染（高斯数量超过颜色编码容量时需分批）
- 速度较慢
- 需要修改并恢复高斯颜色

### 适用场景
小规模场景或需要精确匹配时使用。

---

## 方案 3：解析贡献度计算

### 原理
根据 Gaussian Splatting 的渲染公式，直接计算每个高斯对每个像素的贡献度。

2D Gaussian 的渲染公式：
```
C(p) = Σ_g α_g · N(p; μ_2d(g), Σ_2d(g)) · c_g
```

其中像素 p 对高斯 g 的贡献为：
```python
contrib(p, g) = α_g · exp(-0.5 · (p - μ_2d)^T · Σ_2d⁻¹ · (p - μ_2d))
```

### 流程
1. 对每个高斯，计算其 2D 投影均值 μ_2d 和协方差 Σ_2d
2. 对每个像素，计算其与各高斯的 Mahalanobis 距离
3. 贡献度大于阈值 → 像素属于该高斯

### 优点
- **精确的数学推导**
- 不需要深度估计
- 不存在尺度偏移问题

### 缺点
- 计算量大（386万高斯 × 百万像素）
- 需要优化（空间索引、GPU并行等）

### 优化方向
- 使用 R-tree 或 KD-tree 加速空间查询
- 利用 Gaussian 的局部性，只计算附近高斯

### 适用场景
通用场景，尤其适合需要数学精确性时。

---

## 方案 4：差分反向传播

### 原理
利用 `diff_gaussian_rasterization` 的可微性，通过梯度判断高斯对变化的贡献。

### 流程
1. 渲染当前 Gaussian 模型得到图像 A
2. 对比 GT 图像计算损失：`L = ||A - GT||²`
3. 反向传播：`∂L/∂θ` 获得每个高斯参数的梯度
4. 梯度范数大的高斯 → 对变化贡献大

### 优点
- 端到端可微
- 不需要额外的几何计算

### 缺点
- **梯度是间接指标**，不等同于像素覆盖
- 可能受优化器状态影响

### 适用场景
作为辅助验证手段。

---

## 方案 5：尺度校准的深度估计

### 原理
修复 Depth-Anything 深度与 Gaussian 几何之间的系统性尺度偏移后，再做深度一致性检查。

### 流程
1. 使用 Depth-Anything 估计单目深度
2. 计算尺度因子：`s = median(z_knn / z_pix)`（基于多视图一致性）
3. 校准深度：`z_pix_calibrated = s · z_pix`
4. 反投影到 3D，kNN 找高斯，深度一致性检查

### 优点
- 可复用深度估计网络

### 缺点
- 需要额外的尺度校准步骤
- 尺度因子估计可能不准确

### 关键发现
实验表明 Gaussian 深度均值（1.72）与 Depth-Anything 深度均值（2.63）存在约 **0.65** 的比例关系。

### 适用场景
已部署深度估计网络的场景。

## 方案 6：椭圆投影覆盖法

### 原理
将每个 3D Gaussian 投影到图像平面形成 2D 椭圆椭圆，计算像素是否被椭圆覆盖。

### 数学推导

**3D Gaussian**：N(μ, Σ)，其中 μ ∈ ℝ³，Σ ∈ ℝ³ˣ³

**投影到相机坐标系**：
```
μ_cam = T_cw @ [μ_world; 1]
```

**投影到 2D 图像**（针孔相机模型）：
```
u = fx · (μ_cam.x / μ_cam.z) + cx
v = fy · (μ_cam.y / μ_cam.z) + cy
```

**2D 协方差传播**（利用一阶泰勒展开）：
```
Σ_2d = J · R · Σ_3d · R^T · J^T
```
其中 J 是投影变换的雅可比矩阵。

### 判断像素是否在椭圆内

使用 Mahalanobis 距离：
```python
d²(p, g) = (p - μ_2d)^T · Σ_2d⁻¹ · (p - μ_2d)
```

如果 `d² < k²`（k 通常取 2~3），则像素 p 被高斯 g 覆盖。

### 流程
1. 对每个高斯 g：
   - 计算 2D 投影均值 μ_2d
   - 计算 2D 协方差 Σ_2d
   - 生成覆盖掩码（椭圆内为 1）
2. 对每个像素，统计覆盖它的所有高斯
3. 根据变化掩码，筛选出属于变化区域的高斯

### 优点
- **符合 Gaussian Splatting 的几何本质**
- **不需要深度估计**，避免尺度偏移问题
- 数学上精确

### 缺点
- 计算量大（需处理所有高斯）
- 需要正确的相机内参和外参

### 优化方向
1. **空间索引**：使用 R-tree 或 Quadtree 加速椭圆-点查询
2. **GPU 并行**：将高斯投影并行化
3. **剪枝**：只处理可见高斯（结合 visibility_filter）
4. **Rasterization**：利用现有光栅化框架计算覆盖关系

### 适用场景
通用场景，尤其适合避免深度估计误差时。

## 方法对比分析

### 精度对比
```
高 ← 方案2（唯一ID）= 方案3（解析计算）= 方案6（椭圆投影）> 方案4（反向传播）> 方案5（尺度校准）→ 低
```

### 速度对比
```
快 ← 方案1（可见性）> 方案5（深度估计）> 方案3（解析计算）= 方案6（椭圆投影）> 方案4（反向传播）→ 慢
```

### 实现复杂度
```
简单 ← 方案1 < 方案5 < 方案2 < 方案3 = 方案4 = 方案6 → 复杂
```

## 推荐方案

### 首选：方案 6（椭圆投影覆盖法）
- 最符合 Gaussian Splatting 的几何原理
- 不依赖深度估计，避免尺度偏移
- 精度高

### 备选：方案 3（解析贡献度计算）
- 数学上等价于方案 6
- 可以利用 GPU 高效实现

### 快速验证：方案 1（可见性）+ 方案 5（深度估计）
- 先用可见性做粗筛
- 再用深度估计做细筛
